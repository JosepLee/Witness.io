## 一、背景与问题

### 现有版本的不满之处

1. **默认首页是信实链（事件图谱），应该是态势分析（地图）**
2. **模块命名不够专业**："点位包"→ 应改为"态势分析"一类更符合情报系统风格的名称
3. **三个模块之间的逻辑关系在 UI 上看不出来**，客户感受不到"眼见为实"的核心理念
4. **态势分析模块深度不足**，对标安总在 openclaw 中做的 demo，功能差距明显

### 参考：openclaw Demo 的设计亮点

安总通过 openclaw（kimi k2p5）生成的 demo 页面位于：
`.openclaw/workspace/israel-iran-situation-map/us-bases-situation-map.html`
`.openclaw/workspace/US_Middle_East_Bases_Analysis/07_Situation_Map/index.html`

关键差异对比：

| 功能 | 当前 satint-demo | openclaw demo |
|------|-----------------|---------------|
| 首页 | 信实链 | **态势图（地图）** |
| 模块名 | 点位包 | **态势图 / Situation Map** |
| 布局 | 地图+详情（2栏） | **基地列表+地图+详情（3栏）** |
| 能力评分 | 单一战斗力(0-100) | **ACI攻击+DCI防御双维度** |
| 每日战损 | 静态7期历史 | **每日时序数据+折线图** |
| 设施清单 | 装备列表 | **每设施独立损坏%+可点击弹窗** |
| 战略价值 | 无 | **S/A/B三级标注** |
| 状态分类 | 单一战斗力色 | **运行中/受损/被摧毁三态** |

## 二、核心产品理念（不变）

### 眼见为实

> 依托遥感卫星影像验证情报真实性，并结合经过验证的情报，提供对未来的可靠预测。

### 信任金字塔（架构本质）

```
┌─────────────────────────────┐
│    大模型预测报告（预测层）    │  ← 智能预测
│  可信预测 · 置信度量化        │
├─────────────────────────────┤
│    信实链（验证层）           │  ← 情报验证
│  开源情报 + 卫星节点锚定       │
├─────────────────────────────┤
│    遥感影像态势分析（事实层）  │  ← 卫星真相
│  卫星影像 · 不可篡改的事实     │
└─────────────────────────────┘
```

**数据流方向：**
```
遥感卫星影像 → 锚定事件节点 → 信实链加权可信度 → 大模型合成预测
```

## 三、重构方案 B：数据流叙事重构

### 3.1 导航与路由变更

**现在：**
```
/ → 信实链
/site → 点位包
/report → 综合报告
```

**重构后：**
```
/ → 态势分析（原点位包，升级）    ← 默认首页
/chain → 信实链                   ← 从 / 改为 /chain
/report → 智能预测（原综合报告）
```

**导航栏模块标签：**

| 图标 | 中文 | 英文 | 含义 |
|------|------|------|------|
| ◎ | 态势分析 | SITUATION MAP | 卫星真相层 |
| ⬡ | 信实链 | EVENT CHAIN | 情报验证层 |
| ▣ | 智能预测 | INTEL REPORT | AI预测层 |

导航栏副标题加入数据流提示：
```
SATINT · 卫星影像 → 信实验证 → 智能预测
```

### 3.2 模块一：态势分析（重点升级）

**路由：`/`**
**定位：卫星真相层 · 事实基础**

#### 布局变更：2栏 → 3栏

```
┌──────────────┬──────────────────────────┬──────────────────┐
│  左侧基地列表  │      Leaflet 地图          │   右侧详情面板    │
│  240px       │      可缩放/点击标记         │   380px          │
│  含筛选/状态  │                            │   含图表/清单     │
└──────────────┴──────────────────────────┴──────────────────┘
```

#### 数据模型升级（mockData.js）

新增字段到 SITES：
```js
{
  // 原有字段保留...

  // 新增：双维度能力评分
  aci: 78,                    // 攻击能力指数 0-100
  dci: 65,                    // 防御能力指数 0-100

  // 新增：状态三态
  status: 'damaged',          // 'operational' | 'damaged' | 'destroyed'

  // 新增：战略价值等级
  strategicValue: 'S',        // 'S' | 'A' | 'B'

  // 升级：每日时序数据（原 scoreHistory 扩展）
  dailyData: {
    dates: ['11/01','11/02','11/03'],
    aci:    [82, 78, 75],
    dci:    [70, 68, 65],
    damage: [0,  12,  18],
  },

  // 升级：设施列表含损坏百分比
  facilities: [
    { name: '主跑道', damage: 30, verified: true },
    { name: '加固机库', damage: 0, verified: true },
  ],
}
```

#### 左侧基地列表

- 基地条目：状态色点 + 基地名 + 国家/类型
- 状态筛选：全部 / 运行中 / 受损 / 被摧毁
- 点击条目：地图 flyTo + 展开右侧详情

#### 地图升级

- 标记颜色三态：
  - 🟢 `#22c55e` 运行中（operational）
  - 🟡 `#f59e0b` 受损（damaged）
  - 🔴 `#ef4444` 被摧毁（destroyed）
- 标记中心显示 ACI 数值（原来是战斗力分数）
- 切换选中基地时：`flyTo()` 平滑移动

#### 右侧详情面板升级

从上到下：

1. **基地头部**：名称 · 国家 · 类型 · 战略价值徽章(S/A/B)
2. **双维度大评分**：ACI + DCI 并排显示（带颜色）
3. **每日趋势折线图**：ACI/DCI 双线，SVG 绘制（不引入 Chart.js，保持轻量）
4. **受损状态标签**：完好无损 / 轻度 / 中度 / 严重
5. **综合能力雷达图**：保留现有五维（火力/机动/防空/情报/后勤）
6. **卫星影像区**：保留占位+扫描线动效，时间轴切换
7. **设施清单**：每设施 + 独立损坏% 进度条 + 是否卫星验证
8. **来自信实链**（关联事件）：列出验证过该基地的事件节点，点击跳转信实链

### 3.3 模块二：信实链（增强跳转联动）

**路由：`/chain`**（原 `/`）
**定位：情报验证层**

主体功能不变，重点增强：

1. **验证节点来源标签**：节点详情面板显示「关联基地：[基地名]」，点击按钮不只是跳转，而是跳转到 `/` 并携带 siteId 使地图 flyTo
2. **左侧面板加入数据流说明**：「✓ 已被卫星影像锚定的节点将提升整链可信度」
3. **节点脉冲动画**：已验证节点的外圈加 CSS 动画（keyframes pulse）

### 3.4 模块三：智能预测（改名+小调整）

**路由：`/report`**
**中文名：智能预测**
**英文名：INTEL REPORT**

主体功能不变，增强：

1. **信源列表加来源标签**：每条信源标注「卫星验证」或「信实链节点」
2. **置信度说明**：增加一行小字「置信度由信实链加权算法生成」
3. **「验证结果反哺 Agent」按钮**：点击触发动效（绿色脉冲从报告面板流向左侧地图方向）
