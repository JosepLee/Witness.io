# SATINT · 卫星情报验证平台 · 开发设计文档

> 本文档用于指导 Claude Code 开发一个纯前端演示 Demo。
> 核心目的：向客户展示系统的 Agent 设计逻辑，所有数据均为 Mock 假数据。

---

## 一、核心产品理念

### 1.1 眼见为实

> 依托遥感卫星影像验证情报真实性，并结合经过验证的情报，提供对未来的可靠预测。

### 1.2 信实链（最核心创新点）

对于海量杂乱的事件，无法一一手动验证。系统通过算法和大模型，将收集到的事件**先组成事件链**，再对链中节点进行卫星验证——每一个被验证的节点，会**提升整条链的加权可信度**。

**信实链的核心逻辑：**
- 原始事件（新闻、信号、开源情报）→ 大模型聚合为事件链
- 链上节点分别有独立可信度评分（0~1）
- 被卫星影像验证的节点评分高，颜色为绿色
- 未验证节点根据信息源和大模型打分，红→橙→黄→绿渐变
- 对某一节点验证，会通过加权算法提升整条链的综合可信度

### 1.3 Agent 存在于三个核心位置

1. **基地评分与态势评估**：结合点位包影像变化 + 开源情报，自动更新基地战斗力评分
2. **未来热点分析**：结合历史情报与当前态势，预判军事行动热区和时间窗口
3. **交互方式双轨**：UI 操作和对话调用接口均可触发 Agent，效果一致

## 二、技术选型

| 技术 | 用途 |
|------|------|
| React + Vite | 整体框架 |
| React Router | 三模块路由 |
| D3.js | 事件链力导向图 |
| Leaflet | 点位包二维地图 |
| Canvas 2D | 报告模块地球演示（可后期升级 CesiumJS） |
| Mock JSON | 全部数据为假数据，格式固定 |

> **关于 CesiumJS**：报告模块先用 Canvas 2D 占位实现三阶段动画，后期由有 Cesium 经验的同事替换 `GlobeDemo` 组件，其余逻辑不动。

## 四、模块详细设计

---

### 模块一：信实链 · Event Chain（路由 `/`）

**负责人：开发者本人**

#### 界面布局

```
┌──────────────┬──────────────────────────────────────┬─────────────────┐
│  左侧面板     │           图谱主区域                   │  右侧详情面板    │
│  链列表+图例  │      D3 力导向知识图谱                 │  点击节点后展开  │
│  240px       │      可缩放/拖拽                       │  300px          │
└──────────────┴──────────────────────────────────────┴─────────────────┘
```

#### 图谱节点设计

- **节点形状**：圆形，外圈弧形进度条表示可信度百分比
- **节点颜色**：红(0) → 橙(0.5) → 绿(1)，连续渐变
- **验证标识**：卫星验证节点中心显示 `✓`，外圈有脉冲光晕
- **节点标签**：节点上方显示事件 ID，下方显示事件名称（可换行）
- **连接边**：
  - 链内连接：实线，带箭头方向
  - 跨链关联：虚线，颜色更浅

#### 交互行为

- **点击节点**：
  - 高亮该节点所在的整条链（其余链淡出至 10% 透明度）
  - 右侧展开事件详情面板
- **左侧链卡片点击**：整条链高亮，其余链淡出
- **右侧面板内容**：
  - 事件名称、信息源、时间、所属链
  - 可信度评分条（带颜色）
  - 验证状态
  - 事件详情文字
  - **若该节点已验证（verified=true）**：显示按钮「查看点位包影像」→ 跳转 `/site/:siteId`
- **点击空白处**：取消高亮，恢复全图显示

#### 左侧面板内容

- 图例（红→绿渐变条 + 验证说明）
- 各链卡片（链名、加权可信度分数条、已验证节点数/总节点数）
- 「全部显示」按钮
- 全局统计（总事件数、已验证、待验证、跨链关联数）

### 模块二：点位包 · Site Package（路由 `/site` 和 `/site/:siteId`）

**负责人：前端同事 A**

#### 界面布局

```
┌────────────────────────────────────┬──────────────────────┐
│         Leaflet 二维地图            │     右侧基地详情      │
│         军事基地标记点              │     340px            │
│                                    │                      │
├────────────────────────────────────┤                      │
│    底部影像时间轴（日期切换）        │                      │
└────────────────────────────────────┴──────────────────────┘
```

#### 地图设计

- 底图：CartoDB Dark Matter（深色风格，适合军事界面）
- 基地标记：圆形图标，颜色对应战斗力评分（红/橙/绿）
- 标记中心显示战斗力分数数字
- 点击标记 → 右侧详情面板切换到该基地

#### 底部时间轴

- 横向日期卡片列表，点击切换当前显示的影像期次
- 激活日期高亮显示，颜色跟随基地战斗力色

#### 右侧详情面板（从上到下）

1. **基地基本信息**：名称、国家/地区、类型
2. **战斗力大分数**：数字 + 颜色方块
3. **30天趋势折线图**：简单 SVG 折线，含面积填充
4. **综合能力雷达图**：五维（火力/机动/防空/情报/后勤），SVG 绘制
5. **当前影像区**：
   - WMTS 影像展示区（Demo 阶段用占位符 + 扫描线动效模拟）
   - 大模型识别结果文字描述
   - 影像置信度百分比
6. **装备清单**：
   - 每条装备：名称、数量、变化量（+/-）、是否影像核实

#### 路由联动

- 从模块一跳转过来时，URL 携带 `siteId`，自动定位到对应基地并展开详情

### 模块三：综合报告 · Intel Report（路由 `/report`）

**负责人：前端同事 B**

#### 界面布局

```
┌────────────────────────────────────┬──────────────────────┐
│   左侧：地球/地图动态演示           │   右侧：自动报告面板  │
│   阶段指示器 + Canvas 演示          │   360px              │
│   + 控制按钮                       │                      │
└────────────────────────────────────┴──────────────────────┘
```

#### 地球演示（三阶段动画）

演示按时间轴分三个阶段，点击「启动演示」按钮顺序播放：

**Phase 1：过去事件还原**
- 飞机轨迹从基地出发，动画飞向目标（实线 + 飞机图标）
- 侦察机在目标区域盘旋

**Phase 2：打击确认**
- 目标点出现爆炸动效（红色脉冲扩散圆）
- 标注「卫星影像已确认」

**Phase 3：预测输出（切换到预测模式）**
- 同一基地出现新的预测飞行轨迹（紫色虚线）
- 预测目标点出现虚线圆圈标注
- 配文：「预测：明日05:30再次出击，目标设施B」

#### 右侧报告面板（从上到下）

1. **报告标题 + 时间 + 整体置信度**
2. **支撑信源列表**：每条信源有置信度分数条 + 是否卫星验证标记
3. **分析摘要正文**：流式文字效果（模拟大模型输出，逐字显示）
4. **自动预测提报区**（Phase 3 后显示）：
   - 预测时间、预测事件、置信度
   - 说明：「系统将在预测时间前2小时自动触发卫星拍摄任务」

#### 控制按钮

- 「启动演示」/ 「重新播放」
- 「验证结果反哺 Agent」（Phase 3 后出现，点击触发动效，象征数据回流到信实链）

> **后期升级方向**：将 `GlobeDemo` 组件从 Canvas 2D 替换为 CesiumJS 三维地球，飞机轨迹用 Cesium Entity + 时钟系统驱动，其余面板逻辑完全不变。

## 五、Mock 数据结构

所有数据集中在 `src/data/mockData.js`，三模块共用，禁止在组件内硬编码数据。

### 5.1 事件节点 EVENTS

```js
{
  id: 'A1',              // 唯一ID，格式：链ID + 序号
  label: '事件名称',      // 节点显示名，可含 \n 换行
  chain: 'A',            // 所属链ID
  score: 0.95,           // 可信度 0~1
  verified: true,        // 是否有卫星影像验证
  siteId: 'S1',          // verified=true 时，关联的基地ID（可为 null）
  source: 'OpenSky轨迹', // 信息来源
  date: '2025-11-03 06:12',
  detail: '事件详细描述文字'
}
```

### 5.2 事件边 EDGES

```js
{
  source: 'A1',          // 来源节点ID
  target: 'A2',          // 目标节点ID
  crossChain: true       // 可选，是否跨链连接（渲染为虚线）
}
```

### 5.3 事件链 CHAINS

```js
{
  id: 'A',
  name: '空袭行动链',
  description: '链的一句话描述'
}
```

### 5.4 基地 SITES

```js
{
  id: 'S1',
  name: '基地名称',
  lat: 36.679, lng: 42.447,
  country: '伊拉克',
  type: '空军基地',
  combatScore: 82,             // 战斗力评分 0~100
  scoreHistory: [65,68,...],   // 近7期历史分数（用于趋势图）
  equipment: [
    { name: '装备名', count: 12, change: '+2', verified: true }
  ],
  imagery: [
    { date: '2025-11-03', label: '任务返回后', desc: '识别结果描述', score: 0.91 }
  ]
}
```

### 5.5 报告 REPORT

```js
{
  title: '报告标题',
  subtitle: '副标题',
  generatedAt: '时间字符串',
  confidence: 0.84,             // 整体置信度
  pastEvents: [...],            // 过去事件（用于地球演示）
  predictions: [...],           // 预测事件
  sources: [
    { label: '信源名', confidence: 0.95, verified: true }
  ],
  summary: '报告正文，支持换行'
}
```

### 5.6 工具函数

```js
chainScore(chainId)   // 计算链级加权可信度，返回 0~1
```

## 六、视觉设计规范

### 颜色系统

```css
--bg: #040810          /* 最深背景 */
--surface: #080f1e     /* 卡片背景 */
--surface2: #0d1a2e    /* 次级背景 */
--border: #1a2d45      /* 边框 */
--accent: #f59e0b      /* 主强调色（琥珀） */
--accent2: #0ea5e9     /* 次强调色（天蓝） */
--green: #22c55e       /* 验证成功/高可信 */
--red: #ef4444         /* 低可信/打击目标 */
--text: #e2e8f0        /* 主文字 */
--text-dim: #64748b    /* 次要文字 */
```

### 可信度颜色映射

| 分数区间 | 颜色 | 含义 |
|---------|------|------|
| 0.9~1.0 | `#22c55e` 绿 | 卫星验证 |
| 0.7~0.9 | `#f59e0b` 琥珀 | 高可信 |
| 0.5~0.7 | `#f97316` 橙 | 中可信 |
| 0~0.5   | `#ef4444` 红 | 低可信/待核实 |

### 字体

- 数据/代码/标签：`JetBrains Mono`
- 标题/Logo：`Syne`

### 整体风格

- 深色科技感，军事情报风格
- 所有数字用 Mono 字体
- 边框细（1px），圆角小（3~4px）
- 动效克制，不滥用动画

## 九、Claude Code 操作指引

### 启动项目

```bash
npm install
npm run dev
# 访问 http://localhost:5173
```

### 每次开发一个功能时

1. 先阅读本文档对应模块的「详细设计」章节
2. 数据只从 `src/data/mockData.js` 读取，不在组件内硬编码
3. 颜色只用上方定义的 CSS 变量或工具函数 `scoreColor()`
4. 每个模块是独立的页面组件，不要在模块间共享 state（通过 URL 参数传递即可）
